# Структура Google Sheet: калькулятор экипировки НЛО

## Лист 1: Настройка

Назначение: справочник слотов, редкостей, статов и базовых значений.

### Слоты

| slot_id | Название | Роль в системе | Комментарий |
|---|---|---|---|
| engine | Двигатель | Мобильность НЛО | Скорость сбора ресурсов и активного участия в бою |
| steering | Руль | Боевой контур | Не меняет управление игрока, а улучшает наведение, стабильность оружия и боевую эффективность |
| cargo | Грузовой модуль | Экономика боя | Вместимость ресурсов для строительства и апгрейда башен |

### Редкости

| rarity_id | Название | Ранг | Множитель бюджета |
|---|---|---:|---:|
| common | Обычная | 1 | 1.00 |
| uncommon | Необычная | 2 | 1.45 |
| rare | Редкая | 3 | 2.10 |
| epic | Эпичная | 4 | 3.00 |

### Статы

| stat_id | Название | Типовое отображение | Зачем нужен |
|---|---|---|---|
| tower_damage | Урон башен | % / flat | Усиливает защитную часть core loop |
| ufo_damage | Урон НЛО | % / flat | Делает НЛО более активным участником боя |
| move_speed | Скорость НЛО | % | Ускоряет сбор ресурсов и реакцию на угрозы |
| cargo_capacity | Вместимость ресурсов | flat / % | Позволяет дольше строить и апгрейдить башни без возврата к сбору |

## Лист 2: Балансовая таблица

Назначение: список предметов. Минимальный демонстрационный набор — 12 строк: 3 слота x 4 редкости.

Рекомендуемые колонки:

| Колонка | Назначение |
|---|---|
| item_id | Уникальный ID предмета |
| Название | Декоративное название предмета |
| slot_id | ID слота |
| Слот | Читаемое название слота |
| rarity_id | ID редкости |
| Редкость | Читаемое название редкости |
| Стат 1 | stat_id |
| Тип 1 | flat или percent |
| Значение 1 | Число |
| Стат 2 | stat_id |
| Тип 2 | flat или percent |
| Значение 2 | Число |
| Стат 3 | stat_id |
| Тип 3 | flat или percent |
| Значение 3 | Число |
| Стат 4 | stat_id |
| Тип 4 | flat или percent |
| Значение 4 | Число |
| Комментарий баланса | Короткое объяснение роли предмета |

## Лист 3: Визуализация данных

Назначение: проверяющий собирает сет из трех выпадающих списков и видит итоговый прирост.

Блоки:

1. Базовые характеристики НЛО.
2. Выбор предметов: Двигатель, Руль, Грузовой модуль.
3. Итоговые статы: базовое значение, плоский бонус, процентный бонус, итог, прирост.
4. Короткое резюме выбранного сета.

## Принцип расчета

Для каждого стата:

`Итог = (Базовое значение + сумма flat-бонусов) * (1 + сумма percent-бонусов)`

Такой расчет легко читать, проверять и расширять. Если понадобится более сложный баланс, можно добавить soft cap или diminishing returns, но для тестового задания лучше оставить модель прозрачной.

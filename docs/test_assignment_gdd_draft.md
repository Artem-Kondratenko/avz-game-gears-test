## 1. Наблюдения после прохождения

### 1.1. Что важно в текущей игре

Aliens vs Zombies: Invasion сочетает несколько действий:

- игрок управляет НЛО в реальном времени;
- собирает ресурсы на карте;
- строит и усиливает башни;
- защищает базу от волн зомби;
- периодически сталкивается с усиленными врагами и боссами;
- развивает НЛО и связанные с ним мета-системы.

Ключевое ограничение для новых боевых сущностей: во время боя игрок не может свободно перестраивать, переносить или уничтожать уже поставленные башни. Поэтому новая механика не должна требовать идеальной расстановки “задним числом”.

### 1.2. Что уже закрывают существующие башни

В игре уже есть башни, закрывающие основные боевые роли:

| Башня | Основная роль |
|---|---|
| Турель | универсальный урон по одной цели |
| Лед. кинжал | быстрые выстрелы и шанс заморозки |
| Мортира | взрывной урон по области |
| Арктический шквал | конусная атака по группе врагов |
| Электро-рельсотрон | мощный дальний урон по одиночной цели |
| Черная дыра | замедление и постоянный урон в области |
| Огненная баллиста | дальний урон по одной цели несколькими снарядами |
| Тесла | цепная молния против групп врагов |
| Лазер | нарастающий урон по сильной цели |
| Снайпер | редкий, но мощный урон по крепким врагам |
| Лёдомет | взрывной урон с замедлением |

### 1.3. Где есть место для новой механики

Уже хорошо представлены:

- прямой урон;
- урон по области;
- замедление;
- заморозка;
- дальний урон;
- борьба с группами и одиночными целями.

Слабее представлены:

- временные окна уязвимости;
- фокус по элитной цели без ручного микроконтроля;
- поддержка уже построенной обороны без требования ставить башни рядом;
- механика, которая помогает против босса, но не превращается в еще один источник чистого урона.

Отсюда основная идея новой башни: она должна не просто стрелять сильнее, а помогать игроку лучше реализовать уже построенную оборону.

# Задание 1. Новая башня и новый босс

---

## 2. Новая башня: Маяк Уязвимости

### 2.1. Краткое описание

**Маяк Уязвимости** — башня поддержки против боссов и элитных врагов.

Она почти не решает волну собственным уроном. Ее ценность в другом: башня выделяет самую опасную цель, накладывает на нее статус сканирования и открывает короткое окно, в которое другие башни и НЛО наносят больше пользы.

Коротко:

- роль: анти-босс / анти-элита;
- тип атаки: сканирующий луч;
- собственный урон: низкий;
- основной эффект: временное окно уязвимости на сильной цели;
- главная польза: помогает уже построенным башням лучше реализовать урон.

### 2.2. Почему эта башня нужна

В игре уже есть сильные башни против одиночных целей: Лазер, Снайпер, Рельсотрон, Огненная баллиста. Но они часто работают как отдельные источники урона.

Маяк Уязвимости добавляет новый тип решения:

> “Я не просто строю еще одну пушку, я создаю момент, когда уже построенная оборона становится сильнее против конкретной угрозы.”

Это хорошо подходит игре, где башни не всегда стоят рядом и не могут быть переставлены во время боя.

### 2.3. Механика работы

Башня проходит несколько состояний:

1. **Выбор цели**
   - приоритет: босс;
   - затем элитный враг;
   - затем враг с самым высоким текущим здоровьем.

2. **Сканирование**
   - башня удерживает луч на выбранной цели;
   - на цели появляется заметный визуальный маркер;
   - игрок понимает, что эта цель сейчас важна.

3. **Метка**
   - после короткого накопления цель получает статус “подсвечена”;
   - статус сам по себе не наносит большой урон.

4. **Окно уязвимости**
   - на короткое время цель получает больше урона от башен и НЛО;
   - эффект лучше работает, когда у игрока уже есть несколько разных источников урона;
   - на боссах эффект слабее, но дольше виден, чтобы механика была понятной.

### 2.4. Взаимодействие с существующими башнями

| Башня | Как помогает Маяк Уязвимости |
|---|---|
| Лазер | помогает эффективнее реализовать нарастающий урон по сильной цели |
| Снайпер | делает редкие мощные выстрелы ценнее |
| Рельсотрон | усиливает роль дальнего одиночного урона |
| Огненная баллиста | помогает быстрее прожечь крепкую цель |
| Черная дыра | удерживает цель дольше в зоне обстрела |
| Лёдомет / Лед. кинжал | замедление продлевает время, пока цель получает пользу от окна |
| Тесла / Мортира | меньше прямой синергии, но помогают зачищать окружение вокруг отмеченной цели |
| НЛО | игрок получает понятную причину активно добивать отмеченную цель |

Важно: башни не обязаны стоять рядом с Маяком. Синергия работает через цель и состояние врага, а не через соседство на карте.

### 2.5. Принципы баланса

Числовые значения в тестовом не нужны, поэтому ниже описаны именно принципы.

**Ограничения:**

- одновременно активна только одна основная метка;
- собственный урон башни низкий;
- эффект против боссов слабее, чем против элиты;
- окно уязвимости не должно позволять мгновенно убивать босса;
- повторное наложение на ту же цель должно иметь задержку или сниженную эффективность.

**Что проверять на балансе:**

- не становится ли башня обязательной во всех сборках;
- не ломает ли она боссов;
- достаточно ли она полезна против элиты;
- понимает ли игрок, почему цель стала получать больше урона;
- есть ли смысл строить ее, если у игрока мало других источников урона.

### 2.6. Визуал

Башня должна читаться как поддержка, а не как очередная пушка.

Визуальные признаки:

- сканирующий луч;
- радарная или призматическая форма;
- бирюзово-фиолетовое свечение;
- заметная метка на цели;
- чистый силуэт, понятный на маленьком экране.

Варианты визуала:

1. **Радар-маяк**  
   Более технологичный вариант: тарелка-сканер, луч сверху, ощущение анализа цели.

2. **Призма цели**  
   Более “инопланетный” вариант: голографическая призма с символом прицела, меньше похожа на классическую башню.

Для презентации и PDF можно использовать оба варианта как AI concept references, а финальный выбор оставить за командой.

### 2.7. Обратная связь игроку

Чтобы механика была понятной, нужны три читаемых сигнала:

- луч Маяка показывает, кого башня выбрала;
- иконка/контур на враге показывает, что цель просканирована;
- смена цвета или вспышка показывает начало окна уязвимости.

Если визуальный сигнал будет слабым, игрок воспримет башню как “непонятную слабую пушку”. Поэтому читаемость эффекта важнее, чем сложность анимации.

## 3. Новый босс: Панцирный Мутант

### 3.1. Краткое описание

**Панцирный Мутант** — mid-game босс, который учит игрока не полагаться только на один тип урона.

Он не требует перестраивать башни, но заставляет адаптироваться во время боя: следить за фазой панциря и менять активный фокус между башнями, НЛО и контролем.

Коротко:

- роль: обучающий босс средней стадии;
- пассивная способность: адаптивный панцирь;
- активная способность: сброс панциря;
- главная задача: стимулировать игрока менять способ атаки во время боя.

### 3.2. Пассивная способность: Адаптивный панцирь

Босс отслеживает, какой источник урона доминирует.

Если игрок слишком долго наносит основной урон одним способом, босс временно получает к нему сопротивление.

Примеры:

- если игрок слишком сильно опирается на одиночный урон, босс получает сопротивление к одиночным атакам;
- если основной вклад идет от НЛО, босс временно снижает эффективность урона НЛО;
- если игрок держит босса только контролем и замедлением, босс получает сопротивление к контролю.

Важно: сопротивление не должно полностью отключать стратегию игрока. Оно должно мягко снижать эффективность и подталкивать к другому действию.

### 3.3. Активная способность: Сброс панциря

Через определенные промежутки боя босс сбрасывает текущий слой брони и меняет фазу.

Перед сменой фазы нужен короткий телеграф:

- панцирь начинает светиться;
- цвет панциря меняется;
- над полосой здоровья может появляться простой значок фазы.

После смены фазы игроку выгодно поменять фокус:

- подключить НЛО;
- дождаться окна Маяка Уязвимости;
- использовать башни с другим типом воздействия;
- зачищать мелких врагов, если босс временно плохо пробивается.

### 3.4. Фазы босса

| Фаза | Что означает | Как игрок отвечает |
|---|---|---|
| Синий панцирь | выше сопротивление к контролю | использовать прямой урон |
| Желтый панцирь | выше сопротивление к одиночному урону | подключить урон по области и НЛО |
| Фиолетовый панцирь | выше сопротивление к НЛО | держать фокус башнями |

Цвета можно изменить на этапе внедрения. Главное — не конкретный цвет, а сама логика: фаза должна быть понятна без чтения длинных подсказок.

### 3.5. Как босс заставляет адаптироваться

Панцирный Мутант не ломает стратегию игрока, а проверяет ее гибкость.

Игрок должен:

- следить за цветом панциря;
- понимать, какой источник урона сейчас менее эффективен;
- активнее использовать НЛО в подходящие моменты;
- ценить башни поддержки и контроля;
- не строить всю оборону только вокруг одного способа нанесения урона.

### 3.6. Принципы баланса

**Что важно:**

- босс должен быть понятен с первой встречи;
- сопротивления не должны быть слишком жесткими;
- фаза должна длиться достаточно долго, чтобы игрок успел среагировать;
- активная смена фазы должна иметь заметный телеграф;
- босс подходит для середины игры, поэтому не должен требовать знания всех систем.

**Риски:**

- если сопротивление слишком сильное, игрок решит, что его башни “сломались”;
- если телеграф слабый, игрок не поймет, почему урон просел;
- если фаз слишком много, босс станет перегруженным;
- если фазы слишком короткие, игрок не успеет адаптироваться.

### 3.7. Визуал

Визуальный образ:

- крупный мутант с массивным панцирем;
- цветные светящиеся швы на броне;
- читаемый силуэт на вертикальном поле боя;
- не хоррор, а яркий стилизованный босс в духе мобильной игры.

Варианты визуала:

1. **Мутант-танк**  
   Более гуманоидный, крупный, агрессивный. Хорошо читается как сильный босс.

2. **Панцирная форма**  
   Более низкий, тяжелый, похожий на бронированного мутанта. Лучше подчеркивает механику панциря.

# Задание 2. Калькулятор экипировки НЛО

---

## 4. Общая идея системы

В игре уже есть НЛО как активный участник боя и мета-прогрессия, связанная с его развитием. Поэтому система экипировки НЛО выглядит естественным расширением текущей структуры.

Цель системы:

- дать игроку понятную настройку НЛО под стиль игры;
- связать боевую часть, сбор ресурсов и строительство башен;
- дать дизайнеру простой инструмент для проверки суммарных бонусов.

Для тестового задания система сделана компактной: не сотни строк предметов, а минимальный набор, который показывает принцип работы.

### 4.1. Слоты

| Слот | Роль | Что усиливает |
|---|---|---|
| Двигатель | мобильность | скорость НЛО, активность на поле, частично урон НЛО |
| Руль | боевой контур | не меняет управление, а улучшает боевую эффективность и наведение |
| Грузовой модуль | экономика боя | вместимость ресурсов для строительства и усиления башен |

Важно: “Руль” не должен менять само управление игрока. Лучше трактовать его как стабилизатор боевого контура: точность, наведение, согласование НЛО и башен.

### 4.2. Редкости

| Редкость | Роль в системе |
|---|---|
| Обычная | показывает базовую функцию предмета |
| Необычная | усиливает основную роль предмета |
| Редкая | добавляет второй полезный вектор |
| Эпичная | дает сильный бонус и может иметь небольшую компенсацию |

Редкость повышает не просто числа, а “бюджет” предмета: чем выше редкость, тем больше статов и тем шире влияние на сборку.

### 4.3. Выбранные характеристики

| Характеристика | Почему выбрана |
|---|---|
| Урон башен | напрямую влияет на защитную часть игры |
| Урон НЛО | делает НЛО более активным участником боя |
| Скорость НЛО | влияет на сбор ресурсов и реакцию игрока |
| Вместимость ресурсов | влияет на строительство и усиление башен |

Эти характеристики выбраны потому, что они связаны с основным игровым циклом: НЛО собирает ресурсы, игрок строит башни, башни и НЛО вместе отбиваются от волн.

### 4.4. Принцип расчета

Формула расчета:

```text
Итог = (Базовое значение + сумма плоских бонусов) × (1 + сумма процентных бонусов)
```

Почему так:

- формула простая;
- ее легко проверить вручную;
- она подходит для Google Sheet;
- дизайнер видит вклад плоских и процентных бонусов отдельно.

### 4.5. Пример предметов

| Предмет | Слот | Редкость | Роль |
|---|---|---|---|
| Пульсарный двигатель | Двигатель | Обычная | базовая скорость и небольшой вклад НЛО |
| Пульсарный двигатель Mk II | Двигатель | Редкая | скорость, урон НЛО и небольшой бонус башням |
| Боевой руль Азимут | Руль | Обычная | универсальный боевой прирост |
| Боевой руль Азимут Mk II | Руль | Редкая | баланс между башнями и НЛО |
| Компрессионный трюм | Грузовой модуль | Обычная | больше ресурса, небольшая цена в скорости |
| Компрессионный трюм Mk II | Грузовой модуль | Редкая | экономика боя и умеренный урон |

Полный демонстрационный набор находится в Google Sheet.

- расчет итоговых характеристик;
- краткое резюме выбранного набора.

- скрин/изображение интерфейса;
- краткое объяснение принципа расчета.

# Задание 3. Распределение игроков по турнирным комнатам

## 5. Общая идея турнира

Турнир длится 48 часов: субботу и воскресенье.

Игроки соревнуются в виртуальных комнатах по 50 человек. Очки начисляются за уничтожение зомби на уровнях.

Главная идея:

> Игрок играет в привычный основной режим, но получает дополнительную цель на выходные: подняться выше по башне турнирных комнат и получить более ценные награды.

Турнир не должен требовать отдельного сложного режима. Он должен усиливать уже понятный игровой цикл.

### 5.1. Почему формат комнат подходит игре

Комнаты дают несколько преимуществ:

- игрок видит ограниченную и понятную группу соперников;
- проще создать ощущение “я могу догнать”;
- можно разделить игроков по силе;
- можно сделать рост через повышение комнаты;
- награды легче привязать к уровню комнаты.

---

## 6. Башня виртуальных комнат

Визуальная метафора турнира — хай-тек башня комнат.

Нижние комнаты:

- стартовые;
- более мягкая конкуренция;
- базовые награды.

Средние комнаты:

- игроки, которые уже показали хороший результат;
- конкуренция плотнее;
- награды выше.

Верхние комнаты:

- сильные и активные игроки;
- самые ценные награды;
- выше престиж.

Такой образ легко объясняет игроку прогресс:

> “Побеждай равных, поднимайся выше, получай призы лучше.”

## 7. Алгоритм распределения

### 7.1. Основной параметр: Мощь

В игре есть параметр **Мощь**. Его можно использовать как основу честного первичного распределения.

Почему Мощь подходит:

- игрок уже видит этот параметр;
- он отражает общий уровень развития;
- его легче объяснить, чем скрытую формулу;
- он помогает не смешивать новичков и сильных игроков в одной стартовой комнате.

### 7.2. Дополнительные параметры

Помимо Мощи стоит учитывать:

| Параметр | Зачем нужен |
|---|---|
| Активность за последние 7 дней | чтобы комнаты были живыми |
| Результат прошлого турнира | чтобы успешные игроки поднимались выше |
| Время входа в событие | чтобы поздние игроки не попадали в проигранную комнату |
| Текущая турнирная комната | чтобы сохранять прогресс между турнирами |
| Средние очки за бой | чтобы точнее оценивать реальную силу игрока в событии |

### 7.3. Первичное распределение

1. Игрок входит в турнир.
2. Система смотрит его Мощь.
3. Игрок попадает в диапазон Мощи.
4. Система подбирает комнату до 50 человек внутри этого диапазона.
5. Если комната почти заполнена, открывается следующая.

Так игрок на старте получает соперников, близких по развитию.

### 7.4. Повышение комнат

После завершения турнира игрок может подняться выше.

Пример:

- топ игроков комнаты I переходят в комнату II;
- топ игроков комнаты II переходят в комнату III;
- слабый результат может оставить игрока на текущем уровне или опустить ниже.

Это создает долгосрочную цель:

- не просто выиграть один турнир;
- а постепенно подняться в более ценные комнаты.

### 7.5. Награды

Чем выше комната, тем ценнее призы.

Это важно, потому что без разницы в наградах повышение будет ощущаться декоративным.

Призы можно усиливать по нескольким направлениям:

- больше валюты;
- редкие материалы;
- ускорители;
- сундуки события;
- косметические или статусные элементы;
- прогресс в отдельной шкале выходного турнира.

### 7.6. Защиты от плохого опыта

Нужны ограничения:

- не добавлять позднего игрока в комнату, где лидеры уже сильно оторвались;
- не повышать игрока слишком резко, если он выиграл случайно за счет слабой комнаты;
- не смешивать игроков с огромной разницей в Мощи;
- не делать монетизацию настолько сильной, чтобы подбор казался нечестным.

## 8. AI-промпт для распределения игроков

```text
Ты AI-ассистент турнирного распределения игроков в Aliens vs Zombies: Invasion.

Нужно распределить игроков по виртуальным комнатам до 50 человек.

Цель:
- сохранить ощущение честной конкуренции;
- использовать параметр Мощи как основной критерий первичного распределения;
- учитывать успех игрока в прошлой комнате;
- поддерживать живые комнаты на протяжении 48 часов;
- не отправлять поздних игроков в комнаты, где лидеры уже сильно оторвались.

Данные игрока:
- player_id;
- power: Мощь игрока;
- current_room_tier;
- previous_tournament_rank;
- average_points_per_battle;
- battles_last_7_days;
- sessions_last_7_days;
- event_join_time.

Правила:
1. Новых игроков сначала распределяй по диапазонам Мощи.
2. Игроков, занявших высокие места в прошлой комнате, поднимай на уровень выше.
3. Игроков со слабым результатом не повышай, даже если их Мощь выросла.
4. Не добавляй позднего игрока в комнату, где top-10 уже слишком далеко впереди.
5. Если подходящей комнаты нет, создай новую комнату того же уровня.
6. Чем выше уровень комнаты, тем выше ценность наград.

Верни:
- room_id;
- room_tier;
- player_id;
- краткую причину распределения.
```

## 9. Proof-of-work генератор

В web-презентации должен быть простой генератор, который показывает принцип работы системы:

1. Сгенерировать список игроков.
2. У каждого игрока есть Мощь, активность и прошлый результат.
3. Нажать кнопку распределения.
4. Игроки раскладываются по комнатам.
5. Визуально видно, что более сильные и успешные игроки попадают выше.

Это не production-алгоритм, а понятный прототип для демонстрации логики.

Зачем он нужен:

- показывает, что идея не только описана словами;
- проверяющий может нажать кнопку и увидеть результат;
- демонстрирует умение быстро использовать AI/код для проверки дизайнерской логики.

## 10. Метрики турнира

### 10.1. DAU weekend

Турнир должен увеличить активность в выходные.

Почему:

- событие ограничено по времени;
- игроку есть причина зайти в субботу;
- игроку есть причина вернуться в воскресенье;
- место в комнате может измениться, если игрок не продолжит играть.

### 10.2. Retention

Турнир должен поддержать возвращение игроков.

Почему:

- игрок хочет проверить позицию;
- игрок хочет удержать место;
- игрок хочет подняться в комнату выше;
- башня комнат создает долгосрочную цель на следующие турниры.

Важно: если подбор будет ощущаться нечестным, retention может пострадать. Поэтому справедливое распределение по Мощи — ключевой элемент системы.

### 10.3. ARPU / ARPPU

Турнир может увеличить выручку.

Почему:

- больше сессий повышает шанс покупки;
- игроку понятнее, зачем ему усиление;
- комнаты выше дают более ценные призы;
- можно предложить энергию, бустеры, ускорители или событийные предложения.

Ограничение:

Монетизация не должна разрушать доверие к турниру. Если игрок решит, что победа зависит только от платежей, система может дать краткосрочный рост, но ухудшить удержание.
